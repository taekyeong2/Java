2.로그인 
   로그인창+회원가입창 출력
     -> 아이디 비밀번호 입력받는다.
   if문을 통해 회원정보가 존재(equals)하면 로그인되었다 표시
   아이디는 맞지만 비밀번호 다르면 비밀번호 다시 입력하라 
   (이때 비밀번호 맞고 틀린것을 (reuslt = 1/0 값)
   둘다 틀리면 회원가입하라 -> 회원가입하시겠습니까? 출력 -> y 회원가입 insert 메소드 
   (기본으로 데이터베이스 테이블에관리자 아이디 비밀번호 존재 )
  
   (if 문) 로그인된다면 게시글 작성 메뉴 출력 (1. 공지사항 게시판 2. 자유게시판 3. 익명게시판)
       ==> management클래스에서 메뉴

3.게시판
 1)게시글
    (1. 공지사항 게시판 2. 자유게시판 3. 익명게시판 4.종료)
    while - switch 문으로 게시판 종류 선택  
    각 게시판 선택했을때 메뉴 출력
    (1.작성 2. 검색 3뒤로가기(switch의 조건값 변경시킴) )
    1. 작성 insert로 제목이랑 내용 입력받기
    공지사항 게시판은 관리자 아이디일때만 작성수정삭제 되도록
    (if문  - 로그인정보?/ 값이 0일때만 메뉴 출력)
    아니면 관리자 아니라고 출력

    작성할때 글 번호는 입력받지 말고 자동으로 ++되게 못하나?
    삭제, 수정은 글 번호로 update delete  (글번호를 입력받아 선택)

    게시글 검색은 아이디로 SELECT해서 일단제목+글번호 불러오기 
    (1.확인(글보기) 2.수정 3.삭제)
    다시 글번호를 통해 선택
   ( 글 확인할때 댓글도 함께 불러오기)
    게시글 검색할때 아이디는 첨 입력했던 값으로 불러오기
    익명게시판은 SELECT항목에 아이디 X
 
 2)댓글

    글 번호로 글 select한 후(3,검색) 뒤에 댓글 작성 (insert)메뉴 출력 후

(1. 로그인 2. 회원가입 3.종료)
1 로그인 -> 아이디/ 비밀번호 입력
 1) 아이디가 없을때 : 없는 아이디 입니다. 회원가입해주세요 / 다시 메뉴
 2-1) 비밀번호가 다를때 : 비밀번호를 다시 입력해 주세요 / 뒤로가기
 2-2) 비밀번호 같을때 -> 로그인 성공 (메뉴출력)   /// 로그인 성공시 그 로그인한 id계속 불러낼수있나?

2. 회원가입 -> 회원가입 정보 입력
 1) 아이디가 중복되었을때 : 중복된 아이디가 있습니다. 다시 입력해 주세요 /뒤로가기
 2) 회원가입 성공 =>  데이터베이스에 정보 등록

(1. 공지사항 게시판 2.자유게시판 3.익명게시판 4.로그인메뉴로 5.종료)
1. 공지사항게시판 
 1) 관리자(값 0 -> (1.작성, 2.수정 3.삭제. 4.뒤로가기)
                         이때 작성수정삭제는 글번호로 
 2) 일반유저 -> (1. 검색 2. 확인 2.뒤로가기)

각 게시판 클릭 후 
((1.작성 2. 검색 3뒤로가기 )
1. 작성 -> 글제목, 글내용 입력받기
   글번호가 자동 ++ 안된다면 글 번호도 같이 입력
2. 검색
(1. 확인(글보기) 2.수정 3.삭제 4. 뒤로가기)
1. 확인 => 만약 로그인한 아이디 불러올수 있다면 그 아이디로 글 불러오기( SELET)
  이때 글 불러오면서 댓글 같이 내용
  1) (1댓글작성 2댓글삭제 3댓글수정 4.글삭제 5.글수정 6.뒤로가기 )
     만약 현재아이디 작성아이디 비교할 수 있다면 댓글도 똑같이 비교해서 같음 실행 
     
2. 수정 3. 삭제 => 글 번호로 수정 삭제 / 만약 수정 삭제할때 작성아이디 비교해서 같으면 삭제수정가능 다르면 ㄴㄴ
